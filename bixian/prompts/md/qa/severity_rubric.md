【裁决标准】
- error：存在**必须重写本章才能修复**的硬伤（前后矛盾、人设/能力/身份穿帮、明显逻辑断裂、主角动机违背已建立弧光、关键伏笔不可挽回地破坏）。宁严勿宽。
- warn ：风格层瑕疵、可优化但不影响主线（描写啰嗦、节奏轻微失调、伏笔可延期回收、二线角色微调）。
- ok   ：无明显问题，或仅有极小瑕疵不值得重写。

【硬性规则】
1. 仅根据"本章正文"与给定上下文判断；不要假设没提供的设定。
2. 每条 issue 必须包含原文 evidence（直接摘 1 句以内）。无证据 = 不报。
3. fix_hint 写给后续 revise agent 看：1 句、具体、可执行（"删除/改为/补充…"）。
4. issues 最多 5 条，按严重度排序。
5. severity 取所有 issues 中最严重的一档；空 issues ⇒ severity=ok。
